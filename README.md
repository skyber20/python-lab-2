## Как использовать
В корневой папке проекта прописать:
```bash
python -m src.main
```
Чтобы завершить работу командой строки, прописать exit.
Изначально текущая директория - корень проекта

## Реализованные команды
### cd:
```bash
cd <path>
```
Можно указывать как относительные, так и абсолютные пути (.., ., ~, C:\Users...). При отсутствии path перемещает в '~'.


### ls:
```bash
ls -[l] [<path1>, <path2>, ...]
```
Выводит список файлов и папок. С флагом -l выводит подробную информацию (размер, дата изменения, права доступа).

### cat:
```bash
cat [<path1>, <path2>, ...]
```
Выводит содержимое текстовых файлов

### cp:
```bash
cp -[r] [<path1>, <path2>, ...]> <to_path> 
```
Копирует файлы и папки. Без флага -r не копирует директории.

### mv:
```bash
mv [<path1>, <path2>, ...]> <to_path> 
```
Переименовывает или перемещает файлы/папки.

### rm:
```bash
rm -[r] [<path1>, <path2>, ...]>
```
Перемещает файлы/папки в корзину .trash. Если нет -r, то директория не будет удалена 
'.', '/', '..', родительские директории - удалять не будет

### zip/tar:
```bash
zip <path> <archive_path>
```
```bash
tar <path> <archive_path>
```
Создают архив .zip или .tar.gz.

### unzip/untar:
```bash
unzip <archive_path>
```
```bash
untar <archive_path>
```
Распаковывают архив .zip или .tar.gz в текущую директорию.

### grep:
```bash
grep <pattern> <path> -[ir]
```
Ищет строки по шаблону в файле или папке. Флаг -i — игнорирует регистр, -r — ищет рекурсивно по всем файлам в папке.

### history:
```bash
history <N>
```
Выводит min(N, 20) строк. если N не указано, выводит 20 последних команд

## Некоторые особенности:
- Опции можно поместить в любое место после наименования команды (кроме grep. Там строго)
- Что то типа '-окак', '-wefjef', '-test.txt' будут распознаны как опции
- Если в наименовании файла/папки содержатся пробелы, то ее нужно брать в кавычки ('Текстовый документ.txt')
- При успешном выполнении undo последняя команда удаляется
